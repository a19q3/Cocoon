# P1.2i Review Hardening And Evidence Freeze

Last updated: 2026-05-19

## Summary

P1.2i freezes the current Redox authority evidence package for later community
review. It does not add a new launcher assumption. The work hardens the
structured child-result evidence path and documents exactly what P1.2d through
P1.2h prove.

## What Changed

- Structured child-result evidence is now evaluated through shared helper logic
  instead of duplicated inline parsing.
- Negative tests cover malformed structured-result JSON.
- Negative tests cover stdout PASS markers without structured results.
- Negative tests cover missing launcher structured results.
- Negative tests cover missing service-or-blocked structured results.
- Negative tests cover mismatched authority structured-result kinds.

## Evidence Boundary

P1.2i keeps the current authority label unchanged:

```text
authority_mode = redox-enforced-capsule-entrypoint
authority_enforced_for_service = true
production_arbitrary_service = false
```

The final production label remains blocked:

```text
authority_mode = redox-enforced
production_arbitrary_service = true
```

## What Is Frozen For Review

| Area | Frozen result |
| --- | --- |
| Launcher mechanism | Open installed executable before restriction, enter manifest-derived restricted namespace, then launch from inherited executable FD. |
| Authority evidence | Structured child/service result plus exit status, receipt body hash, archive link, stdout/stderr log hash, status/log/audit readback. |
| CLI behavior | Default Redox `run` still fails closed unless `--allow-unenforced-authority` or explicit `--enforce-redox-authority` is used. |
| Scope boundary | No supervisor, `contain` backend, dynamic FD broker, bulk FD passing, pkgar payload migration, or final production label. |

## Negative Evidence Tests

The evidence path must not accept:

- malformed `COCOON_AUTHORITY_RESULT_JSON`
- human-readable PASS markers without structured results
- service-only structured results without the launcher result
- launcher-only structured results without a service or blocked result
- authority evidence with the wrong structured-result kind

These tests are local unit tests. QEMU smoke continues to validate the positive
Redox path end to end.
